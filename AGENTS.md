# Agent notes: remove-background

This repo publishes a single installable skill, `remove-background`, for Claude Code and compatible
agents. It removes image backgrounds locally with rembg. Keep these facts in mind when changing it.

## Layout

- `skills/remove-background/SKILL.md` - the skill itself. This is the **only** file `npx skills add`
  copies onto an installing machine, so the skill MUST stay self-contained: the working command is
  inlined in SKILL.md and does not depend on any other file in this repo.
- `scripts/remove-bg.sh` - an optional convenience wrapper for people who clone the repo. It is not part
  of the install and nothing in SKILL.md may require it.
- `skills-lock.json` - registers the published skill for the agentskills.io `skills` CLI. It is
  generated by the CLI (the `computedHash` is not a plain file sha256), so regenerate it with the CLI
  rather than hand-editing the hash.
- `README.md` / `LICENSE` (MIT) / `CONTRIBUTING.md`.

## The rembg invocation (the load-bearing detail)

```
uvx --python 3.11 --with "numba>=0.59.1" --from "rembg[cpu,cli]" rembg i INPUT OUTPUT.png
```

Do not "simplify" this to `uvx rembg`. Why each piece is required:

- `--python 3.11` + `--with "numba>=0.59.1"`: `rembg` -> `pymatting` -> `numba`, and on Python 3.12/3.13
  uvx backtracks to `numba 0.53` / `llvmlite 0.36`, which only builds on Python <3.10 and fails. Pinning
  3.11 with a modern numba pulls prebuilt wheels and resolves cleanly.
- `rembg[cpu,cli]`: the `cpu` extra adds the `onnxruntime` backend, the `cli` extra adds the `i`/`p`
  file/folder commands. Plain `rembg` errors with "onnxruntime backend not found" / "CLI dependencies
  not installed".
- First run downloads the U^2-Net model (~176 MB) to `~/.u2net/` and caches it; later runs are offline.

If a future rembg release relaxes the pymatting/numba pin so a plain `uvx rembg[cpu,cli]` resolves on
current Python, simplify the command and update SKILL.md, README, and this file together - but verify a
real end-to-end run first (input image -> RGBA PNG with transparent + opaque pixels).

## Verifying a change

Prove any change to the command end-to-end, not just by reading: run it on a real image and confirm the
output is mode `RGBA` with both fully transparent pixels (background removed) and opaque pixels (subject
kept). A green install is not proof the removal works.
